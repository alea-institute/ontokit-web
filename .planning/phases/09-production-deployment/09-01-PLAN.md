<task type="auto">
  <name>Task 3: Configure environment variables and run database migrations</name>
  <files>
    Remote: /home/ubuntu/ontokit-api/.env, /home/ubuntu/ontokit-web/.env.local on 54.224.195.12
  </files>
  <read_first>
    None — remote ops. Read existing .env.local to preserve secrets.
  </read_first>
  <action>
1. Create API .env file with PostgreSQL DSN and auth mode:
   ```bash
   ssh ... 'cat > /home/ubuntu/ontokit-api/.env << EOF
DATABASE_URL=postgresql+asyncpg://ontokit:ontokit-folio-prod@localhost:5432/ontokit
AUTH_MODE=optional
STORAGE_TYPE=filesystem
STORAGE_PATH=/home/ubuntu/ontokit-data/repos
DEBUG=false
EOF'
   ```
   Also create the storage directory:
   ```bash
   ssh ... "mkdir -p /home/ubuntu/ontokit-data/repos"
   ```

2. Update Web .env.local — add AUTH_MODE while preserving existing secrets:
   ```bash
   ssh ... 'grep -q "AUTH_MODE" /home/ubuntu/ontokit-web/.env.local || echo "AUTH_MODE=optional" >> /home/ubuntu/ontokit-web/.env.local'
   ```

3. Run database migrations (alembic):
   ```bash
   ssh ... "cd /home/ubuntu/ontokit-api && .venv/bin/alembic upgrade head"
   ```
   This creates all the tables (projects, ontology_index, pull_requests, etc.)

4. Update the API systemd service to load the .env file:
   ```bash
   ssh ... 'sudo tee /etc/systemd/system/ontokit-api.service > /dev/null << EOF
[Unit]
Description=OntoKit API
After=network.target postgresql.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/ontokit-api
EnvironmentFile=/home/ubuntu/ontokit-api/.env
ExecStart=/home/ubuntu/ontokit-api/.venv/bin/uvicorn ontokit.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=5
Environment=PATH=/home/ubuntu/.local/bin:/usr/bin:/bin

[Install]
WantedBy=multi-user.target
EOF'
   ```
   Reload systemd:
   ```bash
   ssh ... "sudo systemctl daemon-reload"
   ```

5. Verify migrations ran:
   ```bash
   ssh ... "cd /home/ubuntu/ontokit-api && .venv/bin/alembic current"
   ```
