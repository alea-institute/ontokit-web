**Create `ontokit/services/llm/rate_limiter.py`** — Redis-based daily rate limiting per D-06:

```python
RATE_LIMITS: dict[str, int | None] = {
    "owner": None,      # unlimited
    "admin": None,      # unlimited
    "editor": 500,      # COST-03
    "suggester": 100,   # COST-04
    "viewer": 0,        # no access
}
```

- `async def check_rate_limit(redis, project_id: str, user_id: str, role: str) -> bool`:
  - Key format: `llm:rate:{project_id}:{user_id}:{YYYY-MM-DD}` (per Pitfall 6 in RESEARCH.md)
  - Use `INCR` + `EXPIRE(86400)` pattern
  - Returns True if within limit, False if exceeded
  - Viewers always return False; owners/admins always return True; None limit = unlimited

- `async def get_remaining_calls(redis, project_id: str, user_id: str, role: str) -> int | None`:
  - Returns None for unlimited (admin/owner), 0 for viewer, or (limit - current_count) for editor/suggester
  - Key format same as above; GET instead of INCR

Per D-06: BYO-key calls still count against rate limit (but NOT budget).

**Create `ontokit/services/llm/budget.py`** — Monthly budget enforcement per D-17, COST-01, COST-02:

- `async def get_monthly_spend(db, project_id: str) -> float`:
  - SQL: `SELECT SUM(cost_estimate_usd) FROM llm_audit_logs WHERE project_id = :pid AND is_byo_key = False AND created_at >= date_trunc('month', NOW() AT TIME ZONE 'UTC')`
  - Returns 0.0 if no rows (per Pitfall 5)

- `async def get_daily_spend(db, project_id: str) -> float`:
  - Same pattern but `created_at >= date_trunc('day', NOW() AT TIME ZONE 'UTC')`
  - Only non-BYO calls

- `async def check_budget(db, project_id: str, config: ProjectLLMConfig) -> tuple[bool, str | None]`:
  - If `config.monthly_budget_usd` is None: return (True, None) — unlimited
  - Check daily sub-cap first (per Open Question 3 in RESEARCH.md): if daily_spend >= daily_cap_usd, return (False, "daily_cap_reached")
  - Then check monthly: if monthly_spend >= monthly_budget_usd, return (False, "budget_exhausted")
  - Return (True, None) if within both limits

- `async def get_budget_status(db, project_id: str, config: ProjectLLMConfig) -> dict`:
  - Returns: monthly_spent_usd, monthly_budget_usd, budget_consumed_pct (0.0-1.0+), burn_rate_daily_usd (avg of last 7 days), budget_exhausted (bool), daily_spent_usd, daily_cap_usd

**Create `ontokit/services/llm/audit.py`** — Audit logging per D-08, LLM-07:

- `async def log_llm_call(db, project_id, user_id, model, provider, endpoint, input_tokens, output_tokens, cost_estimate_usd, is_byo_key) -> LLMAuditLog`:
  - Creates and flushes an LLMAuditLog row
  - Metadata only — no prompt or response content stored (per D-08)

- `async def get_usage_summary(db, project_id: str) -> LLMUsageResponse`:
  - Aggregates: total_calls, total_cost_usd for current month
  - Per-user breakdown: user_id, calls_today, calls_this_month, cost_this_month_usd, is_byo_key
  - burn_rate_daily_usd: average daily cost over the last 7 days
  - budget_consumed_pct: monthly_spent / monthly_budget if budget set, else None

**Create `ontokit/api/routes/llm.py`** with the following routes (per RESEARCH.md API Routes section):

1. **`GET /api/v1/projects/{project_id}/llm/config`** (owner/admin/editor/suggester)
   - Query `ProjectLLMConfig` by project_id
   - Return `LLMConfigResponse` with `api_key_set = bool(config.api_key_encrypted)` — NEVER return the key itself (per LLM-02)
   - 404 if no config exists yet

2. **`PUT /api/v1/projects/{project_id}/llm/config`** (owner/admin only per D-10)
   - Accept `LLMConfigUpdate` body
   - If `api_key` is provided: encrypt with `encrypt_secret()` and store in `api_key_encrypted`
   - If `base_url` is provided and provider is local (ollama/lmstudio/llamafile/custom): validate with `validate_base_url(url, allow_private=True)`. For cloud providers: validate with `validate_base_url(url, allow_private=False)` and enforce HTTPS.
   - Upsert into `project_llm_configs` table
   - Return updated `LLMConfigResponse`

3. **`POST /api/v1/projects/{project_id}/llm/test-connection`** (owner/admin only)
   - Read config from DB, decrypt API key (or use BYO key from `X-BYO-API-Key` header if present)
   - Call `provider.test_connection()`
   - Return `{"success": true}` or `{"success": false, "error": "specific message"}`
   - Timeout: 10 seconds

4. **`GET /api/v1/projects/{project_id}/llm/usage`** (owner/admin only per D-14)
   - Call `get_usage_summary(db, project_id)`
   - Return `LLMUsageResponse` with per-user breakdown

5. **`GET /api/v1/projects/{project_id}/llm/status`** (any authenticated user)
   - Return `LLMStatusResponse`:
     - `configured`: True if ProjectLLMConfig exists and has api_key or is local provider
     - `budget_exhausted`: call `check_budget()` — True if monthly budget reached
     - `daily_remaining`: call `get_remaining_calls()` based on user's role
     - Budget and spend fields from `get_budget_status()`

6. **`GET /api/v1/llm/providers`** (public — no auth required)
   - Return list of `LLMProviderInfo` from `PROVIDER_DISPLAY_NAMES`, `PROVIDER_REQUIRES_KEY`, `PROVIDER_ICON_NAMES`

7. **`GET /api/v1/llm/known-models`** (public — no auth required)
   - Return list of `LLMKnownModel` from `KNOWN_MODELS` dict in registry

**Route authorization pattern** — follow the embeddings route pattern:
- Use `get_current_user` dependency for authenticated routes
- Use `get_project_member` to verify project membership + role
- Owner/admin check: `if member.role not in ("owner", "admin"): raise HTTPException(403)`
- LLM access check: `if not check_llm_access(member.role): raise HTTPException(403)`

**BYO key header handling** (per D-05):
- `test-connection` route reads `request.headers.get("X-BYO-API-Key")`
- Key is NEVER written to any DB column
- Key is NEVER logged in any log statement (per Pitfall 4)

**Budget exhaustion response** (per COST-02):
- When `check_budget()` returns False: return HTTP 402 with `{"detail": "LLM budget exhausted for this month"}`
- This is advisory — actual enforcement happens at dispatch time (Phase 13), but the status endpoint reveals the state now
