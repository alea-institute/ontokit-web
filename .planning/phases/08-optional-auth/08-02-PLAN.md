<objective>
Make the ontokit-web frontend auth optional via AUTH_DISABLED env var.

Purpose: When AUTH_DISABLED=true, the Next.js app should start without requiring Zitadel environment variables, provide an anonymous session so pages render correctly, hide sign-in UI, and let visitors browse ontology content. Existing edit/commit/PR buttons should already be hidden since the anonymous user has "viewer" role (canEdit=false).

Output: New auth-mode helper, modified env validation, modified auth.ts with anonymous provider, modified components to hide auth UI when disabled.
</objective>
