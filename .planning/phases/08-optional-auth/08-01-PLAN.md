<objective>
Make the ontokit-api backend support three auth modes via AUTH_MODE env var:
- `required` (default): Current behavior — all endpoints require valid Bearer tokens
- `optional`: GET endpoints work without auth (OptionalUser returns None), write endpoints still require auth (RequiredUser returns 401). Users CAN sign in via Zitadel to get edit access.
- `disabled`: All endpoints return anonymous user, no Zitadel needed at all

FOLIO will deploy with AUTH_MODE=optional. CatholicOS stays AUTH_MODE=required.
</objective>
