| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` / `slate-900` (dark) | Modal surface (`bg-white dark:bg-slate-900`), entity list rows |
| Secondary (30%) | `slate-100` / `slate-800` (dark) | Modal header, shard row backgrounds, PR group headers, collapsible rows |
| Accent (10%) | `primary-600` (light) / `primary-500` (dark) — `#0284c7` / `#0ea5e9` | Submit button, active drag drop targets, PR group selected state |
| Destructive | `red-600` — `#dc2626` | Discard session button only |

Accent reserved for:
- "Submit suggestions" primary CTA button
- Valid drag drop target highlight ring (`ring-primary-500` — matches existing `tree-item-drop-valid`)
- Active/selected PR group border accent
- Step progress bar active step indicator

Semantic colors (non-accent, non-destructive):
- Shard count badge: `slate-600` background / white text (neutral — not accent, counts are informational)
- "Miscellaneous improvements" shard label: `amber-600` text + `amber-50` background (dark: `amber-900/30`) — visual distinction for catch-all shard
- Cross-cutting shard label: `violet-600` text + `violet-50` background (dark: `violet-900/30`) — visual distinction for cross-ancestry entities
- Success PR links: `emerald-600` text — consistent with AutoSaveStatusBar saved state
- Failed PR row: `red-50` background + `red-600` text for error message — consistent with existing error patterns
- Progress step completed: `emerald-500` check icon — consistent with AutoSaveStatusBar

## Interaction Contract

### ShardPreviewModal — Three Phases

**Phase A: Preview & Adjust**
- Opens immediately when "Submit Suggestions" is clicked and `changesCount > threshold`
- Displays loading skeleton while `cluster()` API call is in-flight (< 500ms expected)
- On cluster response: populates `shardPreviewStore` via `setFromClusterResponse()`, renders nested tree
- Header: summary bar + "Close" (X) button
- Body: scrollable nested PR group → shard → entity tree
- Footer: optional notes textarea (placeholder: "Add notes for reviewers — optional") + "Submit" (primary) + "Cancel" (secondary) buttons
- Escape key: same as Cancel (closes modal, session remains active — no data lost)

**Phase B: Submitting (progress)**
- Triggered when user clicks "Submit" in footer
- Footer buttons hide; progress bar fills modal body
- Steps animate sequentially as each backend operation completes
- Cannot be cancelled mid-flight (Submit button is removed; only close-after-error is offered on partial failure)

**Phase C: Complete**
- Shows success/partial-failure completion screen
- Each PR row: index + GitHub PR link (opens in new tab) + shard count
- Failed PRs: error summary + "Retry failed" button (re-runs `batchSubmit` for failed PR groups only)
- Footer: single "Done" button (primary) — resets session, closes modal

### Drag-and-Drop Interactions

**Entity rows (between shards):**
- `useDraggable` on each entity row; `data: { type: "entity", entityIri, fromShardId }`
- Drag overlay: entity local name truncated to 32 chars, matching `tree-drag-overlay` class style
- Valid drop target: target shard row highlights with `ring-2 ring-primary-500 bg-primary-50 dark:bg-primary-900/20` — matches `tree-item-drop-valid`
- Drop: calls `shardPreviewStore.moveEntity()` atomically
- Keyboard fallback: entity row has "Move to..." dropdown (`@radix-ui/react-dropdown-menu`) listing all other shards

**Shard rows (between PR groups):**
- `useDraggable` on each shard row header; `data: { type: "shard", shardId, fromPrId }`
- Drag overlay: shard label truncated to 40 chars
- Valid drop target: PR group header highlights with `ring-2 ring-primary-500`
- Drop: calls `shardPreviewStore.moveShard()`
- Keyboard fallback: ⋮ menu on shard row → "Move to PR..." submenu

**Entity list ordering within shard (optional):**
- `SortableContext` from `@dnd-kit/sortable` wraps entity list
- Reordering within a shard is cosmetic only — does not affect commit content
- If `@dnd-kit/sortable` is not approved: disable in-shard ordering; button-based interactions still work

### Collapse/Expand

- All PR groups start collapsed by default in large sessions (>3 shards), expanded in small sessions
- All shards start collapsed; user expands to see entity local name list
- Expand/collapse state held in `shardPreviewStore.expandedPrIds` and `expandedShardIds` (both `Set<string>`)
- Chevron icon: `ChevronRight` (collapsed) / `ChevronDown` (expanded) from lucide-react

### Merge / Split

- **Merge:** ⋮ menu → "Merge into..." → submenu listing other shards in same PR group. Confirmation: none (undo not available; user sees result immediately in tree). Summary bar updates reactively.
- **Split:** ⋮ menu → "Split shard" → inline entity checklist appears within shard row (multi-select, no dialog). Selected entities form the new shard. New shard gets label "{parent label} — Part 2". No confirmation required.
- **Neither action is destructive** to suggestion data — all changes are preview-only until Submit.

### Small Session Bypass

- When `changesCount <= SMALL_SESSION_THRESHOLD` (5): skip `ShardPreviewModal` entirely, open existing `SuggestionSubmitDialog` unchanged
- Threshold constant: `SMALL_SESSION_THRESHOLD = 5` in `app/projects/[id]/editor/page.tsx`
- The bypass must apply to both initial submit (`submitSession`) and resubmit (`resubmitSession`) paths

## State Machines

### ShardPreviewModal Internal Phases

```
idle
  │ (Submit button clicked, changesCount > threshold)
  ▼
loading       — cluster() in-flight, skeleton shown
  │ (cluster response received)
  ▼
preview       — user adjusts shards, adds notes
  │ (Submit button in footer clicked)
  ▼
submitting    — progress bar active, footer hidden
  │ (batchSubmit response received)
  ├─ success → complete_success
  └─ partial → complete_partial
```

### Per-PR Step States (ShardSubmitProgressBar)

Each step: `idle` → `active` → `done` | `error`

## Accessibility Contract

- `ShardPreviewModal`: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal title
- Modal title: `id="shard-preview-title"` — `aria-labelledby="shard-preview-title"`
- Close button: `aria-label="Close shard preview"`
- All ⋮ menu triggers: `aria-label="Shard options for {label}"` / `aria-label="Entity options for {local name}"`
- Drag handles: `aria-label="Drag {entity local name} to reorder"` on the drag grip icon
- "Move to..." dropdown: keyboard navigable via Radix DropdownMenu (already accessible)
- Collapse chevron button: `aria-expanded={isExpanded}` + `aria-label="Expand {shard label}"` / `aria-label="Collapse {shard label}"`
- Progress bar region: `role="status"` + `aria-live="polite"` on `ShardSubmitProgressBar` — announces step changes to screen readers
- PR links in success screen: `target="_blank"` + `rel="noopener noreferrer"` + `aria-label="Open PR #{N} on GitHub (opens in new tab)"`
- Focus trap: modal must trap focus within its bounds while open (implement via `@radix-ui/react-focus-trap` or equivalent — Radix Dialog already does this; if custom overlay, use `focus-trap-react` or manual `tabIndex` management)
- Escape key: closes modal (existing `useEffect` pattern from EntityGraphModal)
- Reduced motion: all expand/collapse transitions and drag animations respect `@media (prefers-reduced-motion: reduce)` — inherited from `globals.css`

## Animation Contract

All animations are CSS-only (Tailwind transitions). No JS animation libraries.

| Interaction | Animation | Duration | Easing |
|-------------|-----------|----------|--------|
| Modal open | `opacity-0 → opacity-100` + `scale-95 → scale-100` | 150ms | `ease-out` |
| Modal close | `opacity-100 → opacity-0` + `scale-100 → scale-95` | 100ms | `ease-in` |
| Shard row expand/collapse | `max-height: 0 → auto` via `overflow-hidden` transition | 200ms | `ease-in-out` |
| PR group expand/collapse | Same as shard row | 200ms | `ease-in-out` |
| Progress step activation | `opacity-50 → opacity-100` + left border `bg-primary-600` slide in | 200ms | `ease-out` |
| Drag overlay appear | `opacity-0 → opacity-90` | 80ms | `ease-out` |
| Summary bar number update | No animation — instant update to avoid distraction during user dragging | — | — |

Reduced motion: all durations collapse to 1ms per `globals.css` `@media (prefers-reduced-motion: reduce)` rule (already applied globally — no additional work needed).
