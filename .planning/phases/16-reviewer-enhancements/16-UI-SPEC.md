## Component Inventory

New components this phase builds upon:

### ProvenanceBadge (new)
- Inline badge: `[icon] [label] [confidence%]`
- Icon: Sparkles (llm-proposed, amber-500), Pencil (user-edited-from-llm, blue-500), User (user-written, slate-400)
- Confidence: colored percentage, right of icon+label, dash if null
- Size: 10px text, 12px icon, `px-1.5 py-0.5 rounded-sm` pill
- Only renders inside diff line rows (+lines) — not on deletion or context lines
- Positioned at far right of diff line, same row as the line content

### ShardTabNavigator (new)
- Tab bar above diff view, `role="tablist"` on container
- Each tab: shard label + entity count badge (e.g. "Financial Crimes · 12")
- First tab always: "All" (shows full diff, no shard filter)
- Active tab: `border-b-2 border-primary-500 text-primary-600` — matches existing tab pattern
- Overflow: horizontal scroll (`overflow-x-auto`) when tabs exceed container width, no wrapping
- Max label width per tab: 160px, truncate with ellipsis
- Tab count badge: `rounded-full bg-slate-100 px-1.5 py-0.5 text-xs text-slate-600 dark:bg-slate-700 dark:text-slate-400`

### ShardReviewMarker (new)
- Per-shard action strip at bottom of each shard's filtered diff view
- Three buttons: "Approve shard" (green outline), "Reject shard" (red outline), "Clear" (slate, only visible when marked)
- Approved state shows: CheckCircle icon + "Shard approved" in green-600
- Rejected state shows: XCircle icon + "Shard rejected" + optional feedback snippet in red-600
- When rejected: expands inline textarea for shard-specific feedback (max 500 chars)
- Textarea: `text-sm rounded-md border border-slate-200 dark:border-slate-700 p-2`, resize-none, 3 rows

### SimilarEntitiesInlinePanel (new — per suggestion in diff)
- Collapsible section below each added entity block in the diff
- Trigger: `[ChevronRight] Similar existing entities (N)` — matches SimilarConceptsPanel pattern exactly
- Only renders when duplicate_candidates.length > 0 AND max score > 0.40 (per D-04)
- Candidates list: label + composite score (color-coded per semantic palette)
- Each candidate is clickable — expands inline side-by-side comparison

### DuplicateComparisonExpander (new)
- Triggered by clicking a duplicate candidate in SimilarEntitiesInlinePanel
- Two-column layout: left = proposed entity, right = existing candidate
- Each column: heading ("Proposed" / "Existing"), labels list, comments, annotations, parent classes
- Column gap: 16px; each column min-width 200px, max 50%
- Truncation: annotation values truncate at 3 lines (`line-clamp-3`); "Show more" link if truncated
- Loading state: skeleton rows (h-4 w-full animate-pulse bg-slate-200 dark:bg-slate-700) until lazy fetch resolves
- Border: `rounded-lg border border-slate-200 dark:border-slate-700 p-3 bg-slate-50 dark:bg-slate-900`

### Reused existing components (no modification needed):
- `Button` from `components/ui/button` — action bar (Approve, Reject, Request Changes)
- `RejectSuggestionDialog` — whole-PR rejection (unchanged)
- `RequestChangesDialog` — whole-PR request changes (unchanged)
- Existing `DiffView` diff renderer — extended with provenance badges and similar panels, not replaced

## Interaction States

### Shard Tab Navigation
- Default: "All" tab active, full diff rendered
- Tab switch: immediate client-side filter of diff blocks by shard entity IRIs
- No loading state needed — filtering is synchronous over already-loaded diff data
- Keyboard: arrow keys navigate tabs (standard tablist ARIA pattern)

### ProvenanceBadge visibility
- Always visible (not collapsible) — reviewer must see provenance without extra clicks
- Confidence dash (`—`) when `confidence` field is null

### SimilarEntitiesInlinePanel
- Collapsed by default — reviewer must explicitly expand (avoids cognitive overload)
- Expand/collapse: `ChevronRight` → `ChevronDown`, `aria-expanded` on trigger button
- Lazy fetch fires only on first expand (D-07)
- Loading: single spinner row (h-4 w-4 animate-spin rounded-full border-2 border-primary-200 border-t-primary-600)
- Error: "Failed to load similar entities. Try again." with inline retry link

### DuplicateComparisonExpander
- Clicking second candidate while one is open closes the previous, opens new one (accordion behavior)
- Scroll: if comparison content overflows 400px height, inner scroll within expander

### Per-Shard Review Marking
- Mark does not trigger any API call immediately — buffered in local React state
- Marks are included when reviewer submits final PR-level action (Approve / Reject / Request Changes)
- Visual: marked shards update their tab badge color (green dot = approved, red dot = rejected)
- Tab badge dot: 6px circle, `inline-block ml-1 rounded-full` in green-500 or red-500

### "Create Clean PR" button (stretch — D-11)
- Only renders after at least one shard is marked "rejected"
- Positioned in action bar alongside Approve/Reject/Request Changes
- Uses `variant="outline"` with scissors or git-branch icon (Lucide `Scissors` or `GitBranch`)
- Disabled state + tooltip when no shard is rejected: "Mark at least one shard as rejected to generate a clean PR"

## Animation

- Shard tab switching: no animation (immediate filter swap)
- Collapse/expand of SimilarEntitiesInlinePanel: `transition-all duration-150 ease-in-out` on max-height
- DuplicateComparisonExpander open: fade-in `animate-in fade-in-0 zoom-in-95 duration-100`
- ShardReviewMarker feedback textarea: `transition-all duration-100` on height when revealed
- Reduced motion: all transitions suppressed per existing `@media (prefers-reduced-motion)` rule in globals.css

## Accessibility

- `role="tablist"` on ShardTabNavigator container; `role="tab"` + `aria-selected` on each tab
- `aria-controls` on each tab pointing to its filtered diff panel
- `role="status"` + `aria-live="polite"` on shard mark state change (screen reader announcement)
- ProvenanceBadge: full text ("LLM proposed — 87% confidence") as `aria-label` on the badge element
- SimilarEntitiesInlinePanel trigger: `aria-expanded` + `aria-controls`
- DuplicateComparisonExpander: `role="region"` + `aria-label="Comparison: {proposed label} vs {existing label}"`
- All icon-only buttons have `aria-label`
- Focus management: when a tab is selected via keyboard, focus moves to the associated panel
