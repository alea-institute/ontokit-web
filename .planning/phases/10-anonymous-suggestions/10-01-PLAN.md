    **1. Create `ontokit/api/routes/anonymous_suggestions.py`** with a new APIRouter. All endpoints are gated: if `settings.auth_mode == "required"`, return 403 with "Anonymous suggestions not available". Endpoints:

    - `POST /{project_id}/suggestions/anonymous/sessions` — Create anonymous session
      - No auth required (no `RequiredUser` dependency)
      - Rate limit check: query DB for `SuggestionSession` rows WHERE `is_anonymous=True` AND `created_at > now()-1h` AND client IP matches. Use `request.client.host` for IP. If count >= 5, return 429 "Rate limit exceeded. Try again later."
      - Creates session with `user_id=f"anonymous-{secrets.token_hex(6)}"`, `user_name="Anonymous"`, `user_email=None`, `is_anonymous=True`
      - Returns `AnonymousSessionCreateResponse` with `anonymous_token` from `create_anonymous_token(session_id)`
      - Store client IP in a new `client_ip` column on SuggestionSession (add `client_ip: Mapped[str | None] = mapped_column(String(45), nullable=True)` to the model in Task 1's files — executor should add this)

    - `PUT /{project_id}/suggestions/anonymous/sessions/{session_id}/save` — Save to anonymous session
      - Auth via `X-Anonymous-Token` header (NOT Bearer). Extract and verify with `verify_anonymous_token()`. If invalid/expired, return 401.
      - Delegates to `service.save_anonymous(project_id, session_id, data, verified_session_id)`
      - Body: `SuggestionSaveRequest` (reused from existing schema)
      - Returns `SuggestionSaveResponse`

    - `POST /{project_id}/suggestions/anonymous/sessions/{session_id}/submit` — Submit anonymous session
      - Auth via `X-Anonymous-Token` header. Verify with `verify_anonymous_token()`.
      - Body: `AnonymousSubmitRequest` (has summary, submitter_name, submitter_email, honeypot)
      - **Honeypot check**: If `data.honeypot` (aliased as "website") is not None and not empty string, silently return a fake success response: `AnonymousSubmitResponse(pr_number=0, pr_url=None, status="submitted")`. Do NOT actually create anything. This catches bots.
      - Delegates to `service.submit_anonymous(project_id, session_id, data, verified_session_id)`
      - Returns `AnonymousSubmitResponse`

    - `POST /{project_id}/suggestions/anonymous/sessions/{session_id}/discard` — Discard anonymous session
      - Auth via `X-Anonymous-Token` header.
      - Delegates to `service.discard_anonymous(project_id, session_id, verified_session_id)`
      - Returns 204

    - `POST /{project_id}/suggestions/anonymous/beacon` — Beacon save for anonymous sessions
      - Auth via `token` query parameter (same pattern as existing beacon endpoint)
      - Verify with `verify_anonymous_token(token)`. If invalid, return 401.
      - Body: `SuggestionBeaconRequest`
      - Delegates to existing `service.beacon_save()` (beacon_save already works by session_id lookup, no user ownership check needed — but it uses beacon_token verification. For anonymous sessions, create a beacon_token from the session_id during session creation, same as authenticated flow.)

    **2. Add service methods to `SuggestionService`:**

    - `async def create_anonymous_session(self, project_id: UUID, client_ip: str) -> AnonymousSessionCreateResponse`:
      - Verify project exists (call `_get_project`)
      - Generate `session_id = f"s_{secrets.token_hex(8)}"`, `branch = f"suggest/anonymous/{session_id}"`
      - Create anonymous_token via `create_anonymous_token(session_id)`
      - Create beacon_token via `create_beacon_token(session_id)`
      - Create `SuggestionSession` with `user_id=f"anonymous-{secrets.token_hex(6)}"`, `user_name="Anonymous"`, `is_anonymous=True`, `client_ip=client_ip`
      - Create git branch
      - Return `AnonymousSessionCreateResponse`

    - `async def save_anonymous(self, project_id: UUID, session_id: str, data: SuggestionSaveRequest, verified_session_id: str) -> SuggestionSaveResponse`:
      - Get session, verify `verified_session_id == session.session_id`, verify `session.is_anonymous`
      - Reuse same git commit logic as `save()` but skip user ownership check (anonymous token is the auth)
      - Same git lock pattern per branch

    - `async def submit_anonymous(self, project_id: UUID, session_id: str, data: AnonymousSubmitRequest, verified_session_id: str) -> AnonymousSubmitResponse`:
      - Get session, verify token matches, verify is_anonymous

      - Also update `session.user_name` to `data.submitter_name or "Anonymous"` (so PR shows credit)
      - Call `_create_pr_for_session()` with a mock `CurrentUser(id=session.user_id, name=session.submitter_name or "Anonymous")`

      - Delete branch, update status to discarded
