**Module-level helpers:**

`_parse_json_safe(text: str) -> list[dict]` — robust JSON parser per research pitfall 3:

2. Try `json.loads(cleaned)`, extract `["suggestions"]` if dict

`_normalize_confidence(raw: object) -> float | None` — per research pitfall 4 and D-06:
1. If None, return None
2. Try float(raw) — if > 1.0, divide by 100 (likely 0-100 scale)
3. Clamp to [0.0, 1.0]
4. On TypeError/ValueError, return None (handles "high"/"medium"/etc.)

**`class SuggestionGenerationService`:**

```python
def __init__(self, db: AsyncSession):
    self._db = db
    self._assembler = OntologyContextAssembler(OntologyIndexService(db))
    self._validator = ValidationService(db)
    self._dedup = DuplicateCheckService(db)
```

`async def generate(self, project_id: UUID, branch: str, class_iri: str, suggestion_type: SuggestionType, batch_size: int, provider: LLMProvider, project_namespace: str) -> GenerateSuggestionsResponse`:

Pipeline steps (per research Pattern 3):
1. **Assemble context**: `context = await self._assembler.assemble(project_id, branch, class_iri)`
2. **Build messages**: `messages = PROMPT_BUILDERS[suggestion_type](context, batch_size)`
3. **Call LLM**: `text, input_tok, output_tok = await provider.chat(messages)`
4. **Parse JSON**: `raw_suggestions = _parse_json_safe(text)`
5. **Per-suggestion loop (sequential per research pitfall 5)**:
   - Mint IRI: `iri = mint_iri(project_namespace)`
   - Build entity dict from raw suggestion for validation: `{"iri": iri, "label": raw.get("label", ""), "labels": [{"value": raw.get("label", ""), "lang": "en"}], "parent_iris": [class_iri] if suggestion_type in ("children",) else raw.get("parent_iris", [])}`
   - For "children" type: parent_iris = [class_iri] (the current class is the parent of generated children)
   - For "siblings" type: parent_iris = context["parents"][0]["iri"] if context["parents"] else []
   - For "parents" type: parent_iris = the generated parent's own parents (from raw, or empty)
   - For "annotations" and "edges": skip parent_iris validation (VALID-01 doesn't apply to annotations)
   - Validate: `errors = await self._validator.validate_entity(project_id, branch, entity_dict, project_namespace)`
   - Dedup check: `dedup = await self._dedup.check(project_id, raw.get("label", ""), entity_type="class", parent_iri=...)` — skip for "annotations" and "edges" type
   - Build `GeneratedSuggestion` with all fields populated
6. **Return** `GenerateSuggestionsResponse(suggestions=results, input_tokens=input_tok, output_tokens=output_tok)`

For **annotation** suggestions, parse differently:
- `property_iri` from raw suggestion
- `label` = the annotation value
- `lang` from raw (default "en")
- Skip duplicate check (annotations are not deduplicated)
- Skip VALID-01 (annotations don't need parents)

For **edge** suggestions, parse differently:
- `property_iri` = raw.get("property_iri") — expected "rdfs:seeAlso" or "rdfs:isDefinedBy"
- `target_description` = raw.get("target_description")
- `search_query` = raw.get("search_query")
- Skip duplicate check and VALID-01

**Endpoint 1: `POST /{project_id}/llm/generate-suggestions`**
- Response model: `GenerateSuggestionsResponse`
- Parameters: `project_id: UUID` (path), `body: GenerateSuggestionsRequest`, `x_byo_api_key: str | None = Header(default=None, alias="X-BYO-API-Key")`, `db: AsyncSession = Depends(get_db)`, `current_user: RequiredUser = Depends()`
- Flow (same pattern as llm.py routes):
  1. Get member role via SQL query on ProjectMember table
  2. Check LLM access: `if not check_llm_access(role, is_anonymous=False): raise HTTPException(403)`
  3. Get LLM config: query ProjectLLMConfig for project_id; 404 if not configured
  4. Check budget: `within, reason = await check_budget(db, str(project_id), config); if not within: raise HTTPException(402, detail=reason)`
  5. Check rate limit: `if not await check_rate_limit(redis, str(project_id), current_user.id, role): raise HTTPException(429, detail="daily_rate_limit_exceeded")`
  6. Build provider: `provider = get_provider(config.provider, api_key=byo_key or decrypt(config.encrypted_api_key), base_url=config.base_url, model=config.model)`
  7. Detect namespace: query Project by id, call `detect_project_namespace(project)`
  8. Call service: `svc = SuggestionGenerationService(db); response = await svc.generate(project_id, body.branch, body.class_iri, body.suggestion_type, body.batch_size, provider, namespace)`
  9. Audit log: `await log_llm_call(db, str(project_id), current_user.id, config.model or "default", config.provider.value, "llm/generate-suggestions", response.input_tokens, response.output_tokens, cost, is_byo_key=byo_key is not None)`
  10. `await db.commit()` and return response

  For Redis dependency: use the same pattern as llm.py (import `get_redis` or use `aioredis` dependency). If Redis is not available, `check_rate_limit` fails open (returns True) per Phase 11 design.

**Endpoint 2: `POST /{project_id}/llm/validate-entity`**
- Response model: `ValidateEntityResponse`
- Parameters: `project_id: UUID` (path), `body: ValidateEntityRequest`, `db: AsyncSession = Depends(get_db)`, `current_user: RequiredUser = Depends()`
- Flow (lighter — no LLM call, no budget/rate-limit check):
  1. Get member role (must be at least viewer to validate)
  2. Query Project by id, detect namespace
  3. Build entity dict from request body: `{"iri": body.iri or mint_iri(namespace), "label": body.label, "labels": [{"value": body.label, "lang": body.lang}], "parent_iris": body.parent_iris}`
  4. Call `ValidationService(db).validate_entity(project_id, body.branch, entity_dict, namespace)`
  5. Return `ValidateEntityResponse(valid=len(errors) == 0, errors=errors)`
