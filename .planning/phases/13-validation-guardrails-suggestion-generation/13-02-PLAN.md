**File 1: ontokit/services/context_assembler.py**

Create `OntologyContextAssembler` class per D-01/D-03/GEN-06:

```python
class OntologyContextAssembler:
    """Assembles ~2-4K tokens of ontology context for LLM prompt generation.

    Shared utility across all suggestion types (per D-03).
    """
    def __init__(self, db: AsyncSession):
        self._index = OntologyIndexService(db)

    async def assemble(
        self,
        project_id: UUID,
        branch: str,
        class_iri: str,
        max_siblings: int = 10,
        max_ancestor_annotations: int = 3,
    ) -> dict:
```

The `assemble()` method must:
1. Call `self._index.get_class_detail(project_id, branch, class_iri)` — raise ValueError if None
2. Extract parent IRIs from detail["parent_iris"]
3. For each parent IRI (up to 3): call `get_class_detail` to get parent labels and annotations (cap annotations at `max_ancestor_annotations`)
4. Get siblings: call `get_class_children(project_id, branch, detail["parent_iris"][0])` if parents exist, exclude current class IRI, cap at `max_siblings`
5. Get existing children: call `get_class_children(project_id, branch, class_iri)`
6. Return structured dict:
```python
{
    "current_class": {
        "iri": class_iri,
        "labels": detail["labels"],
        "annotations": detail["annotations"],
    },
    "parents": [{"iri": iri, "label": label, "annotations": [...]}],
    "siblings": [{"iri": iri, "label": label}],
    "existing_children": [{"iri": iri, "label": label}],
}
```

Handle root class (no parents): return empty parents list and empty siblings list.

Create the `ontokit/services/llm/prompts/` package with 5 prompt template modules plus an `__init__.py` with the PROMPT_BUILDERS dispatch dict.

Each module exports: `SYSTEM: str` (system instructions) and `build_messages(context: dict, batch_size: int) -> list[dict[str, str]]` (returns messages list for LLMProvider.chat()).

All prompts follow the generative-folio pattern: system message sets role + constraints + JSON output schema, user message provides ontology context + specific request. All require JSON-only output with no markdown.

    "annotations": annotations.build_messages,
    "parents": parents.build_messages,
    "edges": edges.build_messages,
}

**children.py** (GEN-01 — adapt from generative-folio concept_generation.py SYSTEM_INSTRUCTIONS):
- SYSTEM: "You are an ontology expert specializing in legal knowledge representation aligned with the FOLIO (Federated Open Legal Information Ontology) OWL/RDF/SKOS schema. Your task is to suggest child classes (rdfs:subClassOf children) for the given ontology class. Each child must be a specific, non-redundant legal concept that IS-A subtype of the parent class. Output JSON only. No markdown. No explanation outside the JSON structure."
- User message format: Show current class (label + IRI), parents, existing siblings (up to 10), existing children (up to 10). Request `batch_size` child classes.
- JSON output schema: `{"suggestions": [{"label": "string", "definition": "string — one-sentence legal definition", "confidence": 0.0-1.0, "areas_of_law": ["string"]}]}`

**siblings.py** (GEN-02):

- User message: Show current class, the shared parent, existing siblings. Request `batch_size` siblings.
- JSON output schema: same as children.

**annotations.py** (GEN-03 — adapt from generative-folio translation.py TRANSLATION_SYSTEM_INSTRUCTIONS):

For translations, use the standard legal term a legal professional in that jurisdiction would use in formal documents. Output JSON only."
- Include the FOLIO target languages from translation.py: de-de, en-gb, es-es, es-mx, fr-fr, he-il, hi-in, ja-jp, pt-br, zh-cn.
- User message: Show current class with existing annotations. Request annotations.
- JSON output schema: `{"suggestions": [{"property_iri": "string — e.g. skos:altLabel", "value": "string", "lang": "string|null — BCP-47 tag or null for English", "confidence": 0.0-1.0}]}`
