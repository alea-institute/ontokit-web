**Suggestion hook instances** — create one per relevant section:
- `annotationsSuggestions` = `useSuggestions({ ..., suggestionType: "annotations", onAccepted: handleAcceptAnnotation })`
- `edgesSuggestions` = `useSuggestions({ ..., suggestionType: "edges", onAccepted: handleAcceptDomainRange })`

For properties, `suggestion_type: "edges"` is used for domain/range suggestions because domain/range are IRI-valued structural relationships (per RESEARCH.md Open Question 1 recommendation). The `suggestion_type: "annotations"` is used for label/comment/annotation suggestions.

Common opts: `{ projectId, entityIri: propertyIri, branch: branch ?? "main", canUseLLM: canUseLLM ?? false, accessToken, byoKey }`

**Accept flow:**

1. `handleAcceptAnnotation(suggestion, editedValue)`:
   - Merge into the property's annotation edit state. PropertyDetailPanel already has `editLabels`, `editComments`, etc.
   - If `suggestion.property_iri` ends with `#label` → add to `editLabels`
   - If `suggestion.property_iri` ends with `#comment` → add to `editComments`
   - Otherwise → add to `editAnnotations` (other annotation properties)
   - Call the property's save trigger

2. `handleAcceptDomainRange(suggestion, editedValue)`:
   - If the suggestion context indicates domain → add `suggestion.target_iri ?? suggestion.iri` to `editDomainIris`
   - If range → add to `editRangeIris`
   - PropertyDetailPanel already has domain/range edit state — merge into it
   - Call the property's save trigger

**Section modification pattern** — same as ClassDetailPanel:
For each section that exists in PropertyDetailPanel (Labels, Comments, Annotations, Domain, Range, Relationships), add:
1. `SuggestImprovementsButton` in section header (right-aligned)
2. `SuggestionSkeleton` x3 while loading
3. `SuggestionCard` x N when results arrive
4. Inline error banner on failure

**Domain/Range sections** (per D-11): These replace "Children" from the class panel. They should have the same `SuggestImprovementsButton` pattern, but the suggestion cards display the suggested domain/range class IRI + label. Accept button adds the IRI to the domain or range list.

**Duplicate/validation rendering** (per PROP-04): SuggestionCard already handles `duplicate_verdict` and `validation_errors` display. No additional property-specific logic needed — the same card renders identically for properties as for classes.

**headerActions prop**: Add a `headerActions?: React.ReactNode` prop and render it in the panel's header row (same pattern as ClassDetailPanel), so the layout can pass `BranchNavigator` when the property tree tab is active.
