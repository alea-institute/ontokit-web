**2. `BranchNavigator.tsx`** (per D-08, D-09, UX-02):

```typescript
import type { ClassTreeNode } from "@/lib/ontology/types";

export interface BranchNavigatorProps {
  nodes: ClassTreeNode[];
  selectedIri: string | null;
  onNavigate: (iri: string) => void;
  autoSuggestOnNavigate?: boolean;
  onAutoSuggest?: (iri: string) => void;
}
```

Implementation:
- Find the parent of `selectedIri` by searching `nodes` recursively. Then get the parent's `children[]` array. If `selectedIri` has no parent in the tree (it's a root node), use root-level sibling nodes.
- Compute `currentIndex` = index of `selectedIri` in the sibling list.
- `hasPrev` = `currentIndex > 0`, `hasNext` = `currentIndex < siblings.length - 1`.
- Prev button: `rounded-sm p-1.5 text-slate-500 hover:bg-slate-100 dark:hover:bg-slate-700 disabled:opacity-30` + `<ChevronLeft className="h-4 w-4" />` + `aria-label="Previous class in branch"`
- Next button: same pattern with `ChevronRight` + `aria-label="Next class in branch"`
- Position label between buttons: `text-xs text-slate-400` — `{currentIndex + 1} / {siblings.length}` with `aria-live="polite"` for screen reader announcements
- On navigate: call `onNavigate(targetIri)`, then if `autoSuggestOnNavigate` is true, debounce 800ms before calling `onAutoSuggest?.(targetIri)` (Pitfall 6 defense). Use a `useRef` + `setTimeout` for the debounce, clearing on unmount and on subsequent navigations.
- Container: `flex items-center gap-1`
