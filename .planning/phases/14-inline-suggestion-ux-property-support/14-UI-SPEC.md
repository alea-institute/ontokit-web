## Color

**Primary focal point:** The amber `Sparkles` icon prefix on each `SuggestionCard` is the primary focal point for the new UI surface. It draws the user's eye to AI-generated content across all sections of the detail panel, enabling rapid scanning before review. Every interaction in this phase radiates outward from that amber marker.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `bg-white` / `dark:bg-slate-900` (`rgb(var(--background-start-rgb))`) | Main panel background, editor surfaces |
| Secondary (30%) | `bg-slate-50` / `dark:bg-slate-800` | Suggestion card backgrounds, tree panel, section dividers |
| Accent (10%) | `primary-500` (#0ea5e9) / `primary-600` (#0284c7) | Accept button, active tree selection, focus rings, "Suggest improvements" trigger button |
| AI/Sparkle accent | `amber-500` / `amber-100 bg` | Sparkle icon color, confidence badge tint for 70â€“89%, pending count badge in editor header |
| Destructive | `red-600` / `red-50 bg` | Reject button (icon only, no fill), error banners, validation failure messages |

Accent reserved for:
1. The "Suggest improvements" button (primary variant)
2. Accept action icon button on suggestion cards (check icon, `text-primary-600`)
3. Active/selected tree item background
4. Focus rings on all interactive elements
5. Pending suggestion count badge border

Confidence color coding (from D-03):
- 90â€“100%: `text-green-700 bg-green-100 dark:text-green-400 dark:bg-green-900/30`
- 70â€“89%: `text-amber-700 bg-amber-100 dark:text-amber-400 dark:bg-amber-900/30`
- Below 70%: `text-red-700 bg-red-100 dark:text-red-400 dark:bg-red-900/20`

**Source:** `app/globals.css` `@theme` block for primary palette; `components/ui/button.tsx` for variant colors; `components/editor/AutoSaveAffordanceBar.tsx` for amber/green/red status semantics; `components/editor/LLMRoleBadge.tsx` for badge pattern.

## Component Inventory

### New Components (Phase 14)

#### `SuggestionCard`
Inline card rendered below each section of `ClassDetailPanel` and the property detail panel. Follows the same row anatomy as `AnnotationRow`.

**Anatomy:**
```
[sparkle icon] [content] [confidence badge] [accept âś“] [reject âś—] [edit âśŽ]
```

- Container: `flex items-start gap-2 rounded-md border border-dashed border-slate-200 bg-slate-50 px-3 py-2 dark:border-slate-700 dark:bg-slate-800/50`
- Sparkle marker: `<Sparkles className="h-3 w-3 shrink-0 text-amber-500 mt-0.5" />` â€” always present, visually distinguishes AI content
- Content text: `text-sm text-slate-700 dark:text-slate-300` (same as existing annotation row values)
- Confidence badge: `inline-flex items-center rounded-full px-1.5 py-0.5 text-xs font-medium` with color per confidence thresholds above (inherited `font-medium` from existing badge pattern)
- Accept button: `rounded-sm p-1 text-primary-600 hover:bg-primary-50 dark:text-primary-400 dark:hover:bg-primary-900/20` + `<Check className="h-3.5 w-3.5" />`
- Reject button: `rounded-sm p-1 text-slate-400 hover:text-red-500 hover:bg-red-50 dark:hover:bg-red-900/20` + `<X className="h-3.5 w-3.5" />`
- Edit button: `rounded-sm p-1 text-slate-400 hover:text-slate-700 hover:bg-slate-100 dark:hover:bg-slate-700` + `<Pencil className="h-3.5 w-3.5" />`
- Edit mode (inline text field): `flex-1 rounded-md border border-slate-300 bg-white px-2.5 py-1.5 text-sm focus:border-primary-500 focus:outline-hidden focus:ring-1 focus:ring-primary-500 dark:border-slate-600 dark:bg-slate-700 dark:text-white` â€” spacing values `px-2.5 py-1.5` are inherited from existing codebase input pattern

Keyboard: Tab/Shift+Tab moves between cards, Enter accepts focused card, Backspace/Delete rejects, E opens edit mode.

#### `SuggestionSkeleton`
Placeholder shown while LLM request is in-flight. Uses existing `animate-pulse` pattern.

- Container: same dashed border as `SuggestionCard`
- Content line: `h-4 w-3/4 animate-pulse rounded-sm bg-slate-200 dark:bg-slate-700`
- Badge placeholder: `h-4 w-10 animate-pulse rounded-full bg-slate-200 dark:bg-slate-700`
- Show 3 skeleton cards per section while loading (matches UX for "where results will land")

#### `SuggestImprovementsButton`
Trigger button in each section header of ClassDetailPanel (and property detail panel).

- Variant: ghost + sparkle: `flex items-center gap-1.5 rounded-md px-2.5 py-1.5 text-xs font-medium text-primary-600 hover:bg-primary-50 dark:text-primary-400 dark:hover:bg-primary-900/20` â€” spacing values `px-2.5 py-1.5` are inherited from existing codebase button pattern
- Icon: `<Sparkles className="h-3 w-3" />`
- Label: "Suggest improvements" (per D-05)
- Disabled state when LLM budget exhausted: `opacity-50 cursor-not-allowed` â€” renders `LLMBudgetBanner` in its place

#### `SuggestionScopeToggle`
Inline radio-style toggle rendered next to the "Suggest improvements" button for annotation requests.

- Options: "This class" | "Siblings" | "Descendants"
- Layout: `inline-flex rounded-md border border-slate-200 dark:border-slate-700 text-xs`
- Active option: `bg-primary-100 text-primary-700 dark:bg-primary-900/30 dark:text-primary-300 font-medium`
- Inactive option: `text-slate-500 hover:bg-slate-50 dark:text-slate-400 dark:hover:bg-slate-800`
- Default: "This class" (narrowest scope, least surprising)

#### `PendingSuggestionBadge`
Shown in editor header next to the existing action group. Updates in real time.

- Container: `inline-flex items-center gap-1 rounded-full bg-amber-100 px-2 py-0.5 text-xs font-medium text-amber-800 dark:bg-amber-900/30 dark:text-amber-400`
- Icon: `<Sparkles className="h-3 w-3" />`
- Count: e.g. "5" â€” plain number, no label suffix (fits tight header)
- Tooltip on hover: "5 pending suggestions â€” click to scroll to first"
- Clicking: scrolls detail panel to first `SuggestionCard` with `focus()` triggered

#### `BranchNavigator` (Prev/Next)
Added to `ClassDetailPanel` header row when iterating a branch.

- Container: `flex items-center gap-1` appended to existing header action row
- Prev button: `rounded-sm p-1.5 text-slate-500 hover:bg-slate-100 dark:hover:bg-slate-700 disabled:opacity-30` + `<ChevronLeft className="h-4 w-4" />` â€” `p-1.5` inherited from existing header action button pattern

- Position label: `text-xs text-slate-400` e.g. "3 / 12" between the two buttons
- aria-label: "Previous class" / "Next class"

#### `ClassesPropertiesTabBar`
Tab switcher at top of the tree panel (above search input). Replaces or precedes the existing search row.

- Uses `@radix-ui/react-tabs` (already in `package.json`)
- Tab list: `flex border-b border-slate-200 dark:border-slate-700`
- Tab trigger: `flex-1 px-3 py-2 text-xs font-medium text-slate-500 data-[state=active]:text-primary-600 data-[state=active]:border-b-2 data-[state=active]:border-primary-500 dark:text-slate-400 dark:data-[state=active]:text-primary-400`

#### `SuggestionGroupSection`
Collapsible group for sibling/descendant scope results. Wraps suggestion cards under a class name header.

- Toggle: `<ChevronRight>` / `<ChevronDown>` pattern (matches existing `CrossReferencesPanel`)
- Header: `flex items-center gap-2 text-xs font-medium text-slate-700 dark:text-slate-300 py-1 cursor-pointer hover:text-slate-900`
- Count badge: `text-xs text-slate-400` â€” "(3 suggestions)"

## Interaction States

### Suggestion Card States

| State | Visual |
|-------|--------|
| Default | Dashed border, `bg-slate-50`, sparkle icon amber |
| Hover | `border-solid border-slate-300`, subtle shadow `shadow-sm` |
| Accept focused (keyboard) | `ring-2 ring-primary-500/50 ring-inset` |
| Reject focused (keyboard) | `ring-2 ring-red-400/50 ring-inset` |
| Editing | Text field replaces content text; accept/reject buttons hidden; "Accept" + "Discard edit" appear at right |
| Accepted (flash) | 200ms transition to `bg-green-50 border-green-200 dark:bg-green-900/10 dark:border-green-900/50` before card transitions to normal annotation row |
| Rejected (exit) | Card collapses with `max-height: 0` transition over 150ms, then removed from DOM |

### Loading States

| State | Visual |
|-------|--------|
| Suggestions loading | 3 `SuggestionSkeleton` cards with `animate-pulse`, section header shows `<Loader2 className="h-3 w-3 animate-spin" />` |
| LLM unavailable (no config) | Section header shows tooltip "Configure LLM in project settings" on hover of disabled `SuggestImprovementsButton` |
| Budget exhausted | `LLMBudgetBanner` (existing component) renders inline in section, replaces button |

### Animation Contract

All motion respects `@media (prefers-reduced-motion: reduce)` (set in `globals.css` to `0.01ms`).

- Accepted card flash: `transition-colors duration-200`
- Rejected card collapse: `transition-all duration-150 overflow-hidden`
- Suggestion group expand/collapse: `transition-all duration-150`
- Skeleton shimmer: `animate-pulse` (CSS only, no JS)
- Badge count update: no animation (number changes instantly)

Destructive actions in this phase:
- **Reject suggestion:** Single click on X icon button. No confirmation dialog â€” rejection is reversible within the session (rejected cards are hidden but session state persists). aria-label confirms intent: "Reject suggestion".
- **Discard edit:** When the user clicks "Discard edit" in inline edit mode, the edited text reverts to the original suggestion without confirmation (edit-mode discard is low-stakes; the original suggestion remains visible). Button label "Discard edit" makes the consequence clear without a dialog.
- **Discard all suggestions (session level):** Handled by existing session discard flow (out of scope for this phase's new components â€” deferred to existing `useSuggestionSession` discard path).

## Accessibility Contract

All new components inherit the project's existing a11y baseline from Phase 6C:

- `SuggestionCard` icon buttons: `aria-label` required (no title attribute)
- `SuggestionCard` group: `role="list"` on container, `role="listitem"` on each card
- `SuggestionScopeToggle`: uses `role="radiogroup"` + `role="radio"` with `aria-checked`
- `BranchNavigator`: `aria-label` on both buttons; position label has `aria-live="polite"` so screen readers announce navigation
- `PendingSuggestionBadge`: `aria-live="polite"` region; announces count changes
- `ClassesPropertiesTabBar`: uses Radix Tabs (`role="tablist"`, `role="tab"`, `role="tabpanel"`) â€” built-in from `@radix-ui/react-tabs`
- Keyboard navigation category added to existing `KeyboardShortcutDialog` under heading "Suggestions":
  - Tab / Shift+Tab: Move between suggestion cards
  - Enter: Accept focused suggestion
  - Backspace or Delete: Reject focused suggestion
  - E: Edit focused suggestion
